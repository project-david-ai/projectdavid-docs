# Configuration Reference

Project David Core is configured entirely via environment variables loaded from a `.env` file in the project root. This file is generated automatically on first run ‚Äî you do not need to create it manually.

---

## How Configuration Works

When you run `platform-api --mode up` for the first time, the CLI generates two files:

| File | What it contains |
|---|---|
| `.env` | All environment variables ‚Äî auto-generated secrets, user-supplied keys, and service configuration |
| `docker-compose.yml` | The fully-wired Compose stack referencing those variables |

Both files are created once and left untouched on subsequent runs. To update a specific variable after initial setup without regenerating secrets:

```bash
platform-api configure --set HF_TOKEN=hf_abc123
```

To update multiple variables interactively:

```bash
platform-api configure --interactive
```

## Variable Categories

### Auto-Generated ‚Äî Never Set Manually

These variables are cryptographically generated by the setup script. Deleting `.env` and re-running `platform-api --mode up` will regenerate them safely.

| Variable | Purpose |
|---|---|
| `MYSQL_ROOT_PASSWORD` | MySQL root password |
| `MYSQL_PASSWORD` | MySQL application user password |
| `SECRET_KEY` | Application signing key |
| `DEFAULT_SECRET_KEY` | Default fallback signing key |
| `SIGNED_URL_SECRET` | Signed URL generation secret |
| `SANDBOX_AUTH_SECRET` | Sandbox authentication secret |
| `SMBCLIENT_PASSWORD` | Samba share password |
| `SEARXNG_SECRET_KEY` | SearXNG instance secret |
| `API_KEY` | Internal API key |
| `TOOL_CODE_INTERPRETER` | Stable tool identifier |
| `TOOL_WEB_SEARCH` | Stable tool identifier |
| `TOOL_COMPUTER` | Stable tool identifier |
| `TOOL_VECTOR_STORE_SEARCH` | Stable tool identifier |

---

### User Supplied ‚Äî Required for Full Functionality

These variables are not required to start the stack but related features will be unavailable until they are set.

| Variable | Description | Where to get it |
|---|---|---|
| `HF_TOKEN` | HuggingFace personal access token ‚Äî required for downloading gated models via vLLM or the training stack | [huggingface.co/settings/tokens](https://huggingface.co/settings/tokens) |
| `HF_CACHE_PATH` | Local path where HuggingFace model weights are cached ‚Äî auto-resolved to `~/.cache/huggingface` if left blank | Local filesystem path |

---

### Inference Provider Keys ‚Äî Optional

Leave blank if you are not using these providers. The stack starts and runs without them.

| Variable | Provider |
|---|---|
| `TOGETHER_API_KEY` | TogetherAI |
| `HYPERBOLIC_API_KEY` | Hyperbolic |
| `DEEP_SEEK_API_KEY` | DeepSeek |
| `ADMIN_API_KEY` | Set after bootstrapping admin ‚Äî see [Quick Start](/docs/core-overview) |
| `ENTITIES_API_KEY` | SDK user key |
| `ENTITIES_USER_ID` | SDK user ID |

### Base URLs

All public-facing routes go through the nginx reverse proxy on port 80. These defaults work for local development and should only be changed for production deployments behind a custom domain.

| Variable | Default | Description |
|---|---|---|
| `ASSISTANTS_BASE_URL` | `http://localhost:80` | Public API base URL |
| `SANDBOX_SERVER_URL` | `http://localhost:80` | Sandbox server URL |
| `DOWNLOAD_BASE_URL` | `http://localhost:80/v1/files/download` | File download base URL |
| `HYPERBOLIC_BASE_URL` | `https://api.hyperbolic.xyz/v1` | Hyperbolic API endpoint |
| `TOGETHER_BASE_URL` | `https://api.together.xyz/v1` | TogetherAI API endpoint |
| `OLLAMA_BASE_URL` | `http://ollama:11434` | Internal Ollama endpoint |

### AI Model Configuration

| Variable | Default | Description |
|---|---|---|
| `VLLM_MODEL` | `Qwen/Qwen2.5-VL-3B-Instruct` | Model loaded by the static vLLM service on startup |
| `VLLM_EXTRA_FLAGS` | *(blank)* | Extra flags passed to the vLLM command ‚Äî e.g. `--max-model-len 4096` |

```bash
platform-api --mode up --with-ollama
```

### Training Stack

The training stack is opt-in via `--profile training`.

| Variable | Default | Description |
|---|---|---|
| `TRAINING_PROFILE` | `laptop` | Resource profile ‚Äî `laptop`, `standard`, or `high_end` |
| `RAY_ADDRESS` | *(blank)* | Leave blank to start this node as Ray head. Set to `ray://<head_ip>:10001` to join an existing cluster |
| `RAY_DASHBOARD_PORT` | `8265` | Ray dashboard port ‚Äî accessible at `http://localhost:8265` |

**Training profiles:**

| Profile | Use case |
|---|---|
| `laptop` | Conservative VRAM usage, smaller batch sizes ‚Äî RTX 3060 class and below |
| `standard` | Balanced defaults for mid-range GPUs ‚Äî RTX 3080 / 4070 class |
| `high_end` | Maximises throughput ‚Äî 24GB+ VRAM cards (RTX 3090, 4090, A100) |

## Updating Variables After Setup

To update a single variable:

```bash
platform-api configure --set VARIABLE_NAME=value
```

For example, to set your HuggingFace token:

```bash
platform-api configure --set HF_TOKEN=hf_abc123
```

To update multiple variables interactively:

```bash
platform-api configure --interactive
```

> Never manually edit auto-generated secrets (`MYSQL_ROOT_PASSWORD`, `SECRET_KEY`, etc.) unless you are intentionally rotating credentials across a known deployment.
